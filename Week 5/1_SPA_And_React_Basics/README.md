# SPA and React Basics

## Concept Overview
In a traditional Multi-Page Application (MPA), navigating to a new page involves making a request to the server, downloading a new HTML file, and fully reloading the browser window. This can cause flickering and slower perceived performance.

A Single Page Application (SPA), like this React app, works differently. The browser loads only one initial HTML file. From then on, as you navigate or interact with the app, JavaScript (React) dynamically rewrites the current page's content without ever requiring a full page refresh. This results in a much smoother, app-like experience.

## What this Code Demonstrates
This basic "Hello React" app demonstrates a simple component rendering on the screen. Because it's built with React, any state changes or navigation we might add later would happen instantly on the client side, showcasing the core benefit of SPAs.
